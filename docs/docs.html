Installation

Install ai-nexus globally with a single command:

$ npx ai-nexus install

Or use interactive mode for full control over what gets installed:

$ npx ai-nexus install -i
Requirements: Node.js 18 or higher.

Quick Start

# Interactive setup (recommended)
npx ai-nexus install -i

# Quick install with defaults
npx ai-nexus install

# Use your team's rules
npx ai-nexus install --rules github.com/your-org/team-rules

# Install in current project only
npx ai-nexus init

Browse Marketplace

Open a web-based marketplace to search, install, and remove rules with one click:

$ npx ai-nexus browse        # default port 3847
$ npx ai-nexus browse 8080   # custom port

The marketplace runs locally on http://localhost:3847 and provides:

  • Community tab — browse and search all community rules with category filters
  • My Rules tab — view installed rules and remove with one click
  • Diagnostics tab — system health checks and rule source info
  • Tool status cards — Claude Code, Cursor, Codex at a glance
Local only: The server runs on localhost. No data is sent externally. Install/remove only affects your local files.

How It Works

Claude Code: Semantic Router

A study from ETH Zurich (12 repos, 5,694 PRs) found that loading all rules at once hurts performance by ~3% and increases cost by 20%+. The Semantic Router solves this — a hook runs on every prompt, analyzing what rules you actually need:

  • With AI routing (recommended) — GPT-4o-mini or Claude Haiku analyzes your prompt and picks the right rules. Cost: ~$0.50/month.
  • Without AI (fallback) — keyword matching activates rules based on words in your prompt.
# Enable AI routing
export OPENAI_API_KEY=sk-xxx        # or ANTHROPIC_API_KEY
export SEMANTIC_ROUTER_ENABLED=true

Cursor: Rule Converter

ai-nexus converts .md rules to Cursor's .mdc format, adding description and alwaysApply metadata automatically. After conversion, Cursor's built-in semantic search handles rule filtering — ai-nexus does not run a router for Cursor.

Codex: Aggregated Rules

Individual rule files are aggregated into a single AGENTS.md file, which is loaded at session start. No dynamic loading.

Codex users: select only the rules you need. Since all rules are loaded every session, installing too many wastes tokens. Use the interactive wizard (npx ai-nexus install) to pick only relevant categories and files. Recommended starting set: essential.md, commit.md, security.md.

Update & Local Priority

Rules are installed as independent copies. Your customizations are always safe:

  • Existing files are never overwritten during install or update
  • Only new files from source are added
  • npx ai-nexus update syncs new rules from the latest package
  • Use --force to override (backup first!)
# Sync new rules without overwriting local edits
$ npx ai-nexus update

# Force overwrite everything
$ npx ai-nexus update --force
Migrating from symlink mode? Just run npx ai-nexus update — symlinks are automatically converted to copies.

Semantic Router Setup

By default, ai-nexus uses keyword matching to select rules — no API key needed, zero cost. For smarter rule selection, enable AI-powered routing.

Step 1: Choose a Provider

ProviderModelCost
OpenAIgpt-4o-mini (default)~$0.50/month
Anthropicclaude-3-haiku (default)~$0.50/month

Only one provider is needed. If both keys are set, Anthropic is used first.

Step 2: Set Environment Variables

Add the following to your shell profile (~/.zshrc, ~/.bashrc, or ~/.bash_profile):

# Option A: OpenAI
export OPENAI_API_KEY=sk-xxx
export SEMANTIC_ROUTER_ENABLED=true

# Option B: Anthropic
export ANTHROPIC_API_KEY=sk-ant-xxx
export SEMANTIC_ROUTER_ENABLED=true

Then reload your shell:

$ source ~/.zshrc

Step 3: Custom Model (Optional)

Override the default model with these environment variables:

# Use a different OpenAI model
export OPENAI_MODEL=gpt-4o

# Use a different Anthropic model
export ANTHROPIC_MODEL=claude-3-5-haiku-20241022

Step 4: Verify

Test that the semantic router is working:

$ npx ai-nexus test "write a react component"

Method: AI (Semantic Router)

Selected files (3):
  • rules/essential.md
  • skills/react.md
  • rules/typescript.md

If the output shows AI (Semantic Router), AI routing is active. If it shows Keyword matching, check that your API key and SEMANTIC_ROUTER_ENABLED are set correctly.

Privacy & Security: No telemetry. No analytics. No external data collection. API keys are read from environment variables only — never stored on disk or logged. Your prompts are sent to OpenAI/Anthropic only when semantic routing is explicitly enabled. Without it, keyword-based fallback runs entirely offline.

FAQ

How is this different from Claude Code skills (alwaysApply: false)?

Skills already handle on-demand loading within Claude Code. ai-nexus is for a different use case:

  • You use multiple tools (Claude Code + Cursor + Codex) and want one set of rules
  • You want 230+ community rules without writing everything from scratch
  • You prefer routing via keyword matching (free) or a cheaper model instead of Claude

If you only use Claude Code and skills cover your needs, you may not need ai-nexus.

Why not just put everything in CLAUDE.md or AGENTS.md?

You can — but it doesn't scale. CLAUDE.md loads on every prompt regardless of what you're doing. With 5 rules, that's fine. With 50+, you're burning tokens on Docker best practices while writing a commit message. The ETH Zurich study shows this hurts both performance and cost.

ai-nexus solves this by loading only 2-3 relevant rules per prompt, while keeping the rest parked.

Why not just use Claude Code skills instead of rules?

Skills are great for on-demand workflows you explicitly invoke (/commit, /review). But they don't replace rules — rules are passive guidelines that apply automatically (coding conventions, security standards, naming patterns). You shouldn't have to remember to invoke /security-checklist every time you write code.

ai-nexus handles both: it routes rules and skills based on your prompt, so the right context loads automatically without you thinking about it.
