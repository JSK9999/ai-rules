설치

ai-nexus를 한 줄 명령어로 전역 설치:

$ npx ai-nexus install

또는 대화형 모드로 설치할 항목을 직접 선택:

$ npx ai-nexus install -i
요구사항: Node.js 18 이상.

빠른 시작

# 대화형 설치 (권장)
npx ai-nexus install -i

# 기본값으로 빠르게 설치
npx ai-nexus install

# 팀 규칙 사용
npx ai-nexus install --rules github.com/your-org/team-rules

# 현재 프로젝트에만 설치
npx ai-nexus init

마켓플레이스

웹 기반 마켓플레이스에서 규칙을 검색, 설치, 제거할 수 있습니다:

$ npx ai-nexus browse        # 기본 포트 3847
$ npx ai-nexus browse 8080   # 커스텀 포트

마켓플레이스는 http://localhost:3847에서 로컬로 실행되며 다음을 제공합니다:

  • 커뮤니티 탭 — 카테고리 필터로 모든 커뮤니티 규칙 탐색 및 검색
  • 내 규칙 탭 — 설치된 규칙 확인 및 원클릭 제거
  • 진단 탭 — 시스템 상태 확인 및 규칙 소스 정보
  • 도구 상태 카드 — Claude Code, Cursor, Codex 한눈에 확인
로컬 전용: 서버는 localhost에서 실행됩니다. 외부로 데이터가 전송되지 않습니다. 설치/제거는 로컬 파일에만 영향을 줍니다.

규칙 테스트

주어진 프롬프트에 대해 어떤 규칙이 로딩될지 미리보기:

$ npx ai-nexus test "write a react component with hooks"

Selected rules (3):
  rules/essential.md
  skills/react.md

$ npx ai-nexus test --list    # 등록된 모든 키워드 표시

동작 원리

Claude Code: 시맨틱 라우터

매 프롬프트마다 훅이 실행되어 실제로 필요한 규칙을 분석합니다:

  • AI 라우팅 사용 (권장) — GPT-4o-mini 또는 Claude Haiku가 프롬프트를 분석하고 적절한 규칙을 선택. 비용: 월 ~$0.50.
  • AI 미사용 (폴백) — 키워드 매칭으로 프롬프트의 단어를 기반으로 규칙 활성화.
# AI 라우팅 활성화
export OPENAI_API_KEY=sk-xxx        # 또는 ANTHROPIC_API_KEY
export SEMANTIC_ROUTER_ENABLED=true

Cursor: 규칙 변환기

ai-nexus가 .md 규칙을 Cursor의 .mdc 형식으로 변환하고, description과 alwaysApply 메타데이터를 자동 추가합니다. 변환 후 Cursor의 내장 시맨틱 검색이 규칙 필터링을 처리합니다 — ai-nexus는 Cursor용 라우터를 실행하지 않습니다.

Codex: 통합 규칙

개별 규칙 파일이 하나의 AGENTS.md 파일로 통합되며, 세션 시작 시 로딩됩니다. 동적 로딩 없음.

디렉토리 구조

.ai-nexus/ # ai-nexus 메타데이터 ├── config/ # 모든 소스에서 병합된 규칙 ├── sources/ # 클론된 Git 저장소 └── meta.json # 설치 정보 .claude/ # Claude Code ├── hooks/semantic-router.cjs ├── settings.json ├── rules/ # .ai-nexus/config/에서 복사 ├── commands/ ├── skills/ ├── agents/ └── contexts/ .cursor/rules/ # Cursor (.mdc 파일) ├── essential.mdc └── commit.mdc .codex/AGENTS.md # Codex

업데이트 & 로컬 우선

룰은 독립적인 복사본으로 설치됩니다. 사용자의 커스터마이징은 항상 안전합니다:

  • 기존 파일은 절대 덮어쓰지 않음 (install, update 모두)
  • 소스에서 새 파일만 추가
  • npx ai-nexus update로 최신 패키지의 새 룰을 동기화
  • --force로 강제 덮어쓰기 (백업 먼저!)
# 로컬 수정 보존하면서 새 룰 동기화
$ npx ai-nexus update

# 모든 파일 강제 덮어쓰기
$ npx ai-nexus update --force
symlink 모드에서 마이그레이션? npx ai-nexus update만 실행하면 symlink이 자동으로 복사본으로 변환됩니다.

규칙 형식

규칙은 선택적 프론트매터가 포함된 마크다운 파일입니다:

---
description: 이 규칙을 로딩할 시점 (시맨틱 라우터가 사용)
---

# 규칙 제목

규칙 내용...

카테고리:

디렉토리용도
rules/코딩 규칙 (필수, 보안, 커밋 컨벤션)
commands/슬래시 명령어 (/commit, /review)
skills/도메인 지식 (react, rust, python)
agents/서브 에이전트 (코드 리뷰어, 보안 검사기)
contexts/컨텍스트 파일 (@dev, @research)

팀 규칙

Git을 통해 팀 전체에 규칙을 공유:

# 모든 팀원이 동일한 소스에서 설치
$ npx ai-nexus install --rules github.com/acme/team-rules

# 규칙이 업데이트되면
$ npx ai-nexus update

규칙 저장소 만들기

team-rules/ ├── config/ │ ├── rules/ # 핵심 규칙 │ ├── commands/ # 슬래시 명령어 │ ├── skills/ # 도메인 지식 │ ├── agents/ # 서브 에이전트 │ ├── contexts/ # 컨텍스트 파일 │ ├── hooks/ # semantic-router.cjs │ └── settings.json # Claude Code 훅 설정 └── README.md

멀티 소스 설정

여러 Git 저장소의 규칙을 조합:

# 기본 회사 규칙
$ npx ai-nexus install --rules github.com/acme/base-rules

# 프론트엔드 팀 규칙 추가
$ npx ai-nexus add github.com/acme/frontend-rules

# 보안 규칙 추가
$ npx ai-nexus add github.com/acme/security-rules

# 한 번에 모두 업데이트
$ npx ai-nexus update

커뮤니티 레지스트리

GitHub에서 커뮤니티 기여 규칙을 탐색하고 직접 다운로드 — npm publish 불필요.

# 사용 가능한 모든 규칙 목록
$ npx ai-nexus search

# 키워드로 검색
$ npx ai-nexus search react

# 규칙 다운로드
$ npx ai-nexus get react.md

# 여러 카테고리에 같은 이름이 있을 때 카테고리 지정
$ npx ai-nexus get commit.md --category commands

또는 웹 마켓플레이스를 사용하여 시각적으로 탐색:

$ npx ai-nexus browse
기여하기: 누구나 GitHub 저장소에 PR로 규칙을 기여할 수 있습니다. 머지 후 즉시 사용 가능합니다.
